A necessidade pode ser transmitida pelo próprio usuário

tag <form>
A tag <form> cria formulários que possibilitam que o servidor possa receber informações do usuário.
Dois atributos importantes:
action / method

tag <form> - action
O atributo action:
Quando o formulário for submetido, action vai especificar a URL para onde a informação deve ir 

tag <form> - method
O atributo method pode assumir, normalmente, dois valores: 
get e post
Os dois valores servem para enviar informações obtidas pelo formulário para o servidor

tag <form> - method
get:
Com o get a informação vai na URL do documento. 
Exemplo: o formulário tem o campo “nome”:
 “fulano” é inserido no campo texto
A URL vai ficar: www.dominosite.br/indice.html&nome=fulano

tag <form> - method
post:
Com o post a informação também vai ser passada, mas não vai ser visível no URL
A informação será inserida no corpo da página

Modelos de Banco de Dados
Relacional - Limitações
O modelo relacional foi desenvolvido em torno de 1970 por Edgar F. Codd (pesquisador da IBM)
Contudo, a quantidade de informações que precisam ser armazenadas tem crescido muito (principalmente em aplicações Web) e os bancos relacionais podem sofrer com problemas de escalabilidade*



*estar preparado para crescer / funcionar da mesma forma com mais trabalho

Modelos de Banco de Dados
Não Relacional
A principal motivação para banco de dados não relacionais é a busca para resolução do problema de escalabilidade
São modelados de formas diferentes das relações tabulares usadas nos bancos de dados relacionais
NoSQL (Not only SQL)

O SGBD disponibiliza uma interface para que seus clientes possam incluir, alterar, consultar ou remover  dados previamente armazenados
